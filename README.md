# Jarvis PC Assistant — Starter

Готовый скелет для дипломного проекта «Джарвис на ПК». Минимум работает в **mock-режиме** (без реального микрофона): ты вводишь текст команды, система понимает интент и выполняет действие. Позже подключаются wake-word и ASR.

## 📦 Стек (минимум)
- Python 3.10+
- [optional] `pynput` — глобальные хоткеи (push-to-talk)
- [optional] `pyautogui` — автоматизация (клики/хоткеи/окна)
- [optional] `vosk`, `sounddevice` — офлайн ASR (подключается позднее)

## 🚀 Быстрый старт (mock режим)
```bash
python -m src.jarvis.main --mock
```
Вводи команды типа: `открой браузер`, `сделай тише на 20`, `рабочий режим` и т.д.

## 🎙️ Подключение реального распознавания речи (позже)
1) Установи `vosk`, `sounddevice` и скачай русскую модель Vosk (см. `scripts/notes_vosk.txt`).  
2) Запускай:
```bash
python -m src.jarvis.main --asr vosk
```

## 🧩 Архитектура
- `asr.py` — модуль распознавания речи (MockASR, VoskASR-заглушка).
- `nlu.py` — простое распознавание интентов (шаблоны/регексы).
- `executor.py` — выполнение команд (открыть приложение, громкость, сценарии).
- `wakeword.py` — заглушка wake-word (позже: Porcupine/Vosk KWS).
- `hotkeys.py` — глобальная горячая клавиша «push-to-talk».
- `gui_tray.py` — трей-иконка и настройки (минимальный каркас).
- `main.py` — склеивает всё вместе.

## 🧪 Тест-команды
- «открой браузер»
- «открой vscode»
- «сделай тише на 20» / «сделай громче на 10»
- «рабочий режим» (пример сценария: открыть VS Code и браузер)
- «создай папку Проект»
